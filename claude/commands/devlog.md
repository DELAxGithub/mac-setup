今日の開発日誌を書く/更新する。

## 手順

1. 今日の日付を確認する（YYYY-MM-DD）
2. `devlog/{YYYY-MM-DD}.md` が既にあれば読み込む。なければ新規作成
3. 今回のセッションの作業を把握する:
   - git log --oneline（今日のコミット）
   - 会話内容から開発した内容
4. devlog を書く/更新する（下記フォーマット）
5. `devlog/README.md` があれば、ログ表を更新する（日付・主な開発）
   - Day番号は README.md の既存行から推算する

## devlog フォーマット

```markdown
# Day {N} — {YYYY-MM-DD}

## 開発（3行まとめ）
- 変更1
- 変更2
- 変更3
```

## ルール
- 「開発」は3行以内。非エンジニアにもわかる粒度
- 既存セクションは消さない（追記のみ）
- 事実のみ書く。捏造しない
- devlog以外のファイルは変更しない
- 日本語で書く
- **ユーザーに明示的に呼ばれたときのみ実行する。自動実行・先行実行しない**
