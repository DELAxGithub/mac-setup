マックスタジオの復帰時や、複数端末で開発している際の「同期ズレ」を安全に解消するためのワークフローです。

## ⚠️ マルチエージェント協調チェック（最初に必ず実行）

このワークスペースでは **Claude Code** と **Antigravity** が共存しています。
同期の前に以下を確認してください:

1. `git log --oneline -3` で直近コミットの `Co-Authored-By` を確認
   - `Claude` が含まれていれば Claude Code が直近で作業した
   - その変更内容を把握してから同期を進める
2. `git status` で未コミット変更がある場合:
   - 自分の前回作業なら → 先にコミット
   - 別エージェントの作業途中なら → ユーザーに確認してから進める
3. コミット時のルール:
   - `git add .` は使わない（個別ファイル指定）
   - secrets (.env, API keys) が含まれていないか確認
   - コミットメッセージは英語

詳細: `~/src/.agents/coordination.md`

---

1. **同期状態の調査**
   `~/src` 内の全ディレクトリに対して `git fetch` を実行し、以下の状態をレポートします。
   - [OK]: 最新です。
   - [BEHIND]: リモートに更新があります（Pullが必要）。
   - [AHEAD]: ローカルに未プッシュの変更があります。
   - [DIRTY]: 未コミットの変更があります。
   - [NO UPSTREAM]: リモート追跡が設定されていません。

2. **安全な同期の実行**
   以下のルールに従って一括同期を試みます。
   - 状態が `BEHIND` かつ `CLEAN` の場合: `git pull --rebase` を実行。
   - 状態が `BEHIND` かつ `DIRTY` の場合: `git stash` -> `git pull --rebase` -> `git stash pop` を実行。
   - 状態が `DIVERGED`（分岐）の場合: ユーザーに通知し、手動解決を促します。

3. **実行コマンド**
   詳細な調査スクリプトは `/tmp/sync_survey.sh` にあります（またはこのワークフロー内で定義）。

4. **完了報告**
   同期に成功したリポジトリと、衝突やエラーで手動対応が必要なリポジトリをリストアップして報告します。
