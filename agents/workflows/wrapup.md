---
description: 作業終了時の片付け・ドキュメント更新 (Wrap-up)
---
本日の開発作業を終了する際に、変更内容を自動で要約し、プロジェクトの「やりたいこと.md」や「README.md」に作業履歴と次回への引き継ぎ事項を記録するワークフローです。

1. コマンドで `git status` と `git diff` を実行し、現在までの変更内容（差分）を把握します。
2. 把握した差分から、以下の内容を分かりやすくまとめます。
   - 【完了した作業】: 本日達成した内容を箇条書き。
   - 【次回の残タスク】: 次回やること、未完了の部分、気づいた課題などを箇条書き。
3. プロジェクトフォルダ内に `やりたいこと.md`、`TODO.md`、`README.md` などのドキュメントがあるか探します。（ユーザーが開いている `やりたいこと.md` が最優先です）
4. 見つけたドキュメントの末尾、または適切な「履歴・TODO」セクションに、本日の日付とともにステップ2で作成した要約を追記します。
5. （ユーザーに確認後）変更されたドキュメントやソースコードを `git add`（個別ファイル指定、`git add .` は使わない）し、適切なコミットメッセージで `git commit` を実行して、作業を清潔な状態で終わらせます。
   - コミットメッセージは英語で書く
   - secrets (.env, credentials, API keys) が含まれていないか確認する
   - **必ず未コミット変更を残さない状態で終わる**（別エージェントが次に作業する可能性があるため）
