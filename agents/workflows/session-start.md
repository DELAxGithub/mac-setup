**重要: このワークフローは他のワークフロー（sync, wrapup）より先に実行してください。**

作業を開始する前に、対象リポジトリの状態と他エージェントの痕跡を確認するワークフローです。

## 手順

1. **対象リポジトリの状態確認**
   - `git status` で未コミット変更を確認
   - `git log --oneline -5` で直近のコミット履歴を確認

2. **他エージェントの痕跡チェック**
   - 直近のコミットに `Co-Authored-By: Claude` が含まれている場合、Claude Code が直近で作業したことを通知する
   - 未コミット変更がある場合、別エージェントの作業途中の可能性を警告する
   - 確認コマンド: `git log --oneline -3 --format="%h %s %b" | grep -i "co-authored\|claude\|antigravity"`

3. **警告が出た場合の対応**
   - 未コミット変更あり → ユーザーに「先にコミットするか、変更を破棄するか」を確認
   - 別エージェントの作業途中 → そのリポジトリでの作業を避け、ユーザーに報告

4. **問題なければ作業開始**
   - 現在のブランチ、最新コミット、未コミット変更の有無を簡潔に報告

## マルチエージェント協調ルール

このワークスペース（~/src/）では Claude Code と Antigravity が共存しています。

- **作業前**: 必ず `git status` で未コミット変更がないか確認
- **コミット時**: `git add .` は使わない（個別ファイル指定）
- **コミットメッセージ**: 英語で書く
- **作業後**: 必ずコミットしてクリーンな状態で終わる（未コミット変更を残さない）
- **大きな変更**: ブランチを切って作業する
- **secrets**: .env, credentials, API keys は絶対にコミットしない
- **force push**: しない

詳細は `~/src/.agents/coordination.md` を参照。
