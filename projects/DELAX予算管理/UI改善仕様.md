# DELAX予算ダッシュボード UI改善仕様

## 現状の課題

1. **経費が1セクションにフラット** — 18行が並列で、固定/変動/裁量の区別がない
2. **節税サマリーに「今月いくら使える」がない** — 年間の経費余力はあるが月別の裁量が見えない
3. **実績と予算の区別が弱い** — 青(実績)とグレー(予算)だけで、境界が不明確
4. **俳句パススルーが売上・外注を膨らませる** — 実質の経営判断には邪魔

---

## 改善2: 裁量メーター（ヘッダー直下に追加）

節税サマリーバーを拡張して「今月の裁量」を最も目立つ位置に:

```
[  今月あと ¥XX万 使える  |  年度残り ¥111万  |  目標利益 ¥100万  |  着地 ¥211万  ]
```

### 計算ロジック
```
年間裁量枠 = 着地利益予測 - 目標利益
今月使える = (年間裁量枠 - 今月以降の計画済み経費) / 残り月数
```

### 表示
- 大きな数字で「今月あと ¥XX万」を中央表示
- 裁量枠が少なくなったらバーの色がgreen→orange→redに変化
- 計画済み支出（旅行、機材等）を差し引いた「本当に自由な額」を表示

## 改善3: 実績/予算の視覚的境界

現在の月ヘッダーに「実績確定線」を追加:

- 実績が入っている最後の月の右端に **太い縦線** を引く
- 左側（実績）= 白背景、右側（予算）= 薄いストライプ背景
- これにより「どこまで確定でどこから予測か」が一目瞭然

### CSS追加
```css
.grid-table td.cell-forecast {
  background: repeating-linear-gradient(
    -45deg, transparent, transparent 3px,
    rgba(0,0,0,0.02) 3px, rgba(0,0,0,0.02) 6px
  );
}
.grid-table td.cell-boundary {
  border-right: 3px solid var(--blue);
}
```

## 改善4: 俳句パススルー表示の切替

グリッドヘッダーに「実質表示 / 全表示」トグルを追加:

- **全表示**（デフォルト）: 現状通り、俳句の売上・外注を含む
- **実質表示**: 俳句をネットして1行「NHK俳句（純利益）」として表示
  - 売上セクション: 俳句行 → ¥790,000/年
  - 外注セクション: 俳句要員費行 → 非表示

## 改善5: 月別計画の改善

現在の月カードを強化:

```
┌──────────────────────────┐
│ 7月                       │
│ 固定: ¥33万               │  ← 自動計算
│ 計画: ¥60万 (旅行)        │  ← 手入力
│ ─────────────             │
│ 残り裁量: ¥XX万           │  ← ハイライト
└──────────────────────────┘
```

## 実装優先度

| 優先度 | 改善 | 工数 | インパクト |
|:---:|------|------|-----------|
| 1 | 裁量メーター | 小 | 最大 — これがダッシュボードの存在理由 |
| 2 | 経費3グループ化 | 中 | 大 — 固定/変動の区別が意思決定を助ける |
| 3 | 実績/予算の境界線 | 小 | 中 — 視認性向上 |
| 4 | 俳句トグル | 小 | 中 — 経営判断用の実質ビュー |
| 5 | 月別計画の強化 | 中 | 中 — 計画フェーズで便利 |

## 技術メモ

- gws CLI移行後はグリッドデータをローカルスクリプトから直接取得可能
- UI自体は現在のHTML/CSS/JSをそのまま活かせる（GAS Web App or ローカルサーバー）
- DUMMY_DATA構造の sections 配列を6セクションに拡張するだけで対応可能
